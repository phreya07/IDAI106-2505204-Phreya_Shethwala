# IDAI106-2505204-Phreya_Shethwala
A hospital queue app that reduces waiting stress with real-time updates and calming features. Simple, clean design ensures an easy and comfortable user experience.

# Overview

This project is a prototype of a hospital queue management application designed to reduce patient stress during waiting periods. It integrates real-time queue tracking with relaxation and engagement features to improve the overall patient experience.

# Objective

The objective of this application is to provide a simple and efficient solution that allows patients to monitor their queue status while reducing anxiety through supportive and interactive features.

# Key Features
 Queue Tracking- The app provides real-time updates on the patient’s queue position along with an estimated waiting time. A progress indicator helps users understand how close they are to their turn.

Relaxation and Mood Support- Users can select their current mood and access guided breathing exercises, short relaxation sessions, and calming messages to manage stress effectively.

Distraction Hub- The application includes a variety of engaging content such as nature videos, mini games, and stories. It also offers personalized recommendations based on user preferences and mood.

Notifications- The system sends timely alerts regarding queue updates and upcoming turns. Users can customize notification settings such as sound and vibration.

Additional Features- The app includes options such as favorites, kids mode, voice assistance, emergency priority alerts, and dark mode to enhance usability and accessibility.

Design- The interface follows a clean and modern design approach using soft pastel colors, rounded components, and balanced spacing. The layout is simple and intuitive, making it accessible for users of all age groups.

# User Flow

1. The user logs into the application
2. The dashboard displays current queue status
3. The user engages with relaxation or distraction features while waiting
4. Notifications alert the user when their turn is approaching
5. The user attends the appointment
6. Feedback is provided after the visit

# Target Users
This application is designed for hospital patients, including children, adults, and elderly individuals who experience waiting times before appointments.

# Conclusion
This prototype demonstrates how a thoughtful combination of technology, design, and user-centered features can significantly improve the hospital waiting experience by reducing stress and keeping users informed and engaged.
